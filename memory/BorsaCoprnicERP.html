1. What is an ERP?

An ERP is a system that integrates various business processes and functions across different departments into a unified and centralized platform. The main goal is to streamline processes and information flow within an organization, making data easily accessible.

ERP systems typically include modules for various functions such as finance, human resources, supply chain management , manufacturing, procurement, and more. By consolidating all these functions into a single system.

An ERP often include the following key features:

Centralized Database

One of the fundamental features of ERP systems is the use of a centralized database. All data from various departments and functions are stored in a single, unified database. This ensures data consistency, reduces redundancy, and provides a single source of truth for the entire organization.

Common User Interface

ERP systems often offer a common user interface across different modules. This standardized interface makes it easier for employees to navigate through various functions and modules

Real Time Reporting and Analytics

ERP systems enable real-time reporting and analytics, allowing organizations to access up-to-date information about their operations

Automation of Routine Tasks

ERP systems automate routine and repetitive tasks across different departments. This can include tasks related to data entry, approvals, and other processes.

Scalability

ERP systems prioritize data security. They often include robust access controls to ensure that sensitive information is only accessible to authorized personnel.

Security and Access Controls

Many modern ERP systems offer mobile accessibility, allowing users to access essential information and perform tasks on-the-go. This feature is particularly valuable for employees who need to stay connected and make decisions outside the traditional office environment.

In summary, ERP systems provide a comprehensive solution for organizations to manage and optimize their data resources effectively. These systems bring together various features to improve collaboration, decision-making, and overall operational efficiency.

2. Types of ERP

There are many types of ERP’s but the most commonly used are the following ones:

Type

Definition

Cloud-based ERP

In this model, the ERP software and data are hosted on a cloud platform , and users access the system through the internet. Cloud ERP is flexible, has good scalability and reduces costs.

On-Premises ERP

This is the Traditional ERP where the software is installed and maintained on the company's own servers. The company is responsible for software updates, maintenance and security.

Hybrid ERP

A hybrid ERP is a blend of on premises-erp and cloud-based solutions. The combination of hosting and deployment services vary by provider. These models can provide ERP users the flexibility to migrate between delivery models, or integrate benefits not available to existing implementations.

Open Source ERP

An open source ERP provides source code access, allowing companies to customise and modify the software according to their specific needs.

Industry-specific ERP

Are ERP that are tailored to meet the specific requirements of particular industries.

3. Advantages and disadvantages of an ERP

Advantages:

  • Streamlined processes: ERP systems integrate various business processes, eliminating silos of information and allowing for smoother operations across departments.
  • Improved efficiency: Centralizing data and processes can enhance efficiency by reducing duplication of efforts, automating tasks, and providing real-time information.
  • Better decision-making: Access to accurate and up-to-date information enables informed decision-making at all levels of the organization.
  • Enhanced reporting and analytics: ERPs often offer robust reporting and analytics tools that can help in analyzing trends, forecasting, and identifying areas for improvement.
  • Standardized operations: Implementing an ERP can lead to standardized processes across the organization, ensuring consistency and compliance with best practices.

Disadvantages:

  • Cost: ERP systems can be expensive to implement, including licensing, customization, training, and maintenance costs.
  • Complex implementation: Implementing an ERP requires significant planning, time, and resources. It might disrupt existing workflows during the transition phase.
  • Customization challenges: While ERPs offer flexibility, extensive customization can lead to complexities, making upgrades and maintenance more difficult.
  • User resistance: Employees might face a learning curve and resistance to change when adapting to a new system, affecting productivity initially.
  • Dependency on vendor: Organizations might become dependent on the ERP vendor for updates, support, and customizations, potentially leading to issues if the vendor faces problems or discontinues support for the software.

3. Objectives of an ERP

An ERP aims to integrate and manage an organizations data and processes across various areas such as finance, human resources, inventory, sales, purchasing, etc. They are designed to enhance efficiency, productivity, and decision-making within a company. This enables greater collaboration between departments, facilitates data accessibility, and problem-solving. Moreover, ERPs ensure that the entire company complies with specific industry standards and regulations. They also enable better management and scalability of functions and tasks.

4.1 Katana ERP (Adrián)

Katana is a cloud-based manufacturing ERP, software for small and midsize businesses looking for an affordable as well as flexible solution. Katana manufacturing ERP offers a wide range of features, from inventory management and production planning to sales order management and material purchasing, to streamline end-to-end manufacturing operations. Katana has multiple plans to suit your business size and maturity level. It also provides an implementation package (at an extra fee) to get you up and running quickly.

4.1.1 Configuration

In this case we are talking about a cloud-based ERP, so in this case, there is no necessary installation, but we will have to perform a start-up configuration, that is going to adapt and configure the ERP to our essential needs.

This configuration phase will run automatically at the first start-up of the ERP server.

4.1.2 Functionalities

The software focuses completely on the end-to-end manufacturing process to the exclusion of everything else. For other business processes such as accounting, CRM, time tracking, etc., you must integrate it with other apps. There are some native integrations available for e-commerce and accounting apps, but not other critical functions such as HR.

4.2 Dolibarr (Jan)

This ERP/CRM is designed for managing businesses, composed of independent and optional functional modules. You can configure the application to meet your specific needs by including the modules that best suit you, and even create your own modules easily.

If you want to install it, go to the downloads page and click the link to Sourceforge downloads.

Once here, click on Download Latest Version, currently 18.0.3, and the executable will download to your computer.

4.3 Tryton (Javier)

Tryton is an open source ERP. It is designed to help companies manage various aspects of their business operations, such as accounting, inventory, sales, and more. Tryton is known for its modularity, scalability, and flexibility, making it suitable for a range of businesses, from small and medium-sized enterprises

4.3.1 Configuration

You have two methods to install: you can use docker and pip install from python.

To configure it with docker, you need to do the following commands.

Start a PostgreSQL instance

docker run --name tryton-postgres -e POSTGRES_PASSWORD=mysecretpassword -e POSTGRES_DB=tryton -d postgres

Setup the database

docker run --link tryton-postgres:postgres -e DB_PASSWORD=mysecretpassword -it tryton/tryton trytond-admin -d tryton --all

Start a Tryton instance

docker run --name tryton -p 8000:8000 --link tryton-postgres:postgres -e DB_PASSWORD=mysecretpassword -d tryton/tryton

Start a Tryton cron instance

docker run --name tryton-cron --link tryton-postgres:postgres -e DB_PASSWORD=mysecretpassword -d tryton/tryton trytond-cron -d tryton

4.3.2 Functionalities

  • Parties

On that module you can make parties, associate addresses and create contact mechanisms (Phone, mobile, fax, E-Mail…).

On that module you can manage your products (price/u, type...).

  • Financial

On that module you can manage the payments, invoices, entries, account movements, balance and Reporting.

  • Inventory and Stock

On that module you can manage your inventory, the products you have in stock, where the product is...

  • Purchases

On that module, you can put products to purchase and be managed.

  • Sales

On that module you see the command of customers , their products purchased and quantity, in addition you can do a report of that.

  • TimeSheet

On that module, you can manage the employee time/task.

  • Productions

On that module, you can manage and plan your productions.

  • Projects

In that module you can manage your projects based on how much time is spent, see the progress with a progress bar and create and manage tasks of the projects.

Tryton is compatible with various platforms, facilitating implementation on different operating systems such as Linux, Windows, and macOS.

Tryton facilitates customization and the development of new features due to its modular architecture, module-based approach and because it is open source, making the possibility for businesses to create his own distribution.

  • Active Community

Tryton has an active community of users and developers contributing to ongoing development and improvement.

Our educational center, Nicolau Copèrnic, needs a web application that connects students and companies. What does this mean? Basically, companies will be able to post job offers that students can subscribe to.

Our objective is to make this application as easy to use and functional as possible, implementing all the requirements asked by the client. These requirements will be explained in the following section.

In the following section, we are going to cover all the requirements that we have gathered during all the meetings with our client.

During the meetings, we have asked a lot of questions to our client and by means of the answers to these questions we have formulated the functional and non-functional requirements for our project.

Some of the questions were the following ones:

  • What do you want to store about the students?
  • What do you want to say about the companies?
  • Do you want a user that can control everything inside the application?

8.3 Architecture diagram

In our case, the backend is going to be hosted in an AWS EC2 cloud virtual machine with a Debian based Linux distro installed as an OS. Then, the backend is composed by the following:

  • Java running the Tomcat web server, that hosts our Spring Boot web application with all the necessary architectural layers inside.
  • SQL engine, in our case a MySQL db engine with all the necessary databases and tables.

Moving on to the frontend, in our case we are going to use React framework. This will render in the client, which will reduce the workload inside the server in front of high affluence of users Inside the React application we are going to manage all the connections with the backend API with the Axios Module, that allows us to perform easily and fast API calls via HTTP using different types of HTTP requests. In general we think that all the principal features of the project have been accomplished but during the development due to lack of time we have not been able to implement all the functionalities as we would like so here are some future ideas to be implemented inside of the project:

  • Give the option to the users to provide apart from the DNI, passports
  • Allow companies to have multiple headquarters inside of the same company and allow them to also assign manager responsible users for each one of the head quarters
  • Validate if the company belongs to a specified place based on the postal code
  • Improve the mailing system with a dedicated microservice to reduce the load over the principal web service
  • Add procedures to the database to clean possible garbage of the cache tables
  • Migrate the full frontend to Next.JS or Astro to have profit of the SSR feature of those frameworks to simplify the deployment into docker
